## Notes

- This document is intentionally sanitized. Replace placeholder domains with
  deploy-time configuration rather than committing environment-specific values.
- k6 and smoke tests now expect credentials through `LOGIN_PASS` or
  `BOOTSTRAP_ADMIN_PASSWORD`.
- The CRM bootstrap path requires `BOOTSTRAP_ADMIN_PASSWORD` in production so
  the repo does not ship a default administrator password.

## Validation checklist

1. `/health` and `/ready` return success from both services.
2. CRM pages load customer, order, invoice, product, and storefront data.
3. Product changes in CRM propagate to the shop catalog.
4. Admin config output shows masked database information and no secret values.
5. Frontend responses do not expose internal cluster hostnames.
6. Observability drilldowns open using configured console URLs only.

## Recommended follow-up for each deployment

- Confirm current DB credentials before rollout or restart.
- Verify CORS origins match the intended CRM, shop, and ops domains.
- Re-run targeted pytest coverage and tracked-file greps before publishing docs
  or pushing changes upstream.
