01 / Overview

What the Simulation Lab Does

The Simulation Lab provides a unified control surface for generating realistic application behavior: infrastructure incidents, database stress, demo data, and cross-service distributed traces. Every action produces correlated telemetry across OCI APM, Logging, Log Analytics, DB Management, and Operations Insights.

Chaos Issue Simulation

One-shot incidents (DB latency, error bursts, N+1 queries, app exceptions) and persistent toggles (DB disconnect, slow responses, error rate injection).

Data Demo Data Generation

Generate orders, backlog, high-value anomaly orders, and customers. All writes go to Oracle ATP with traceable source_system=crm-simulation.

Proxy Cross-Service Control

Forward simulation commands to the Drone Shop (C28) via the internal service key. Trigger distributed traces that span CRM + Shop + ATP.

OCI Observability Correlation

Every endpoint creates OTel spans with simulation-specific attributes. Logs include severity spikes. DB queries appear in Performance Hub.

05 / Middleware

Chaos Middleware Pipeline

The ChaosMiddleware runs before the session gate on every request. It reads both frozen config flags (env vars) and runtime-mutable state (_sim_state).

Chaos TypeSourceTriggerInjectionPath Exclusions
CPU Spikecfg.simulate_cpu_spike30% random0.5s busy-loopNone
Memory Leakcfg.simulate_memory_leak100%+1 MB per request (cumulative)None
Slow Queriescfg.simulate_slow_queries100% on /api/*1-5s random sleepNon-API paths
Error Rate_sim_state["error_rate"]Configurable %HTTP 500 JSON response/api/simulate/*

Error rate exclusion: /api/simulate/* paths are always allowed through so the reset/configure endpoints remain reachable during chaos. This is the auto-remediation path that OCI coordinators and k6 teardown use to restore normal operation.

Chaos bypass: All demo data endpoints use async_session_factory() directly instead of the get_db() context manager. This bypasses the chaos simulation hooks (simulate_db_disconnect, simulate_db_latency) so data generation always succeeds regardless of active chaos state.

09 / Security

Authentication & Access Control

Session Gate

/api/simulate/* is NOT in the public path list. All simulation endpoints require a valid session cookie (obtained via POST /api/auth/login or SSO).

Admission Control

Auth lookups run in a dedicated ThreadPoolExecutor with semaphore-based admission control. Overflow returns 503 with Retry-After header.

Proxy Restrictions

Only 5 actions allowed through the drone shop proxy: configure, reset, db-latency, error-burst, status. All others return 403-equivalent error.

Chaos Self-Protection

error_rate injection excludes /api/simulate/* paths so the reset endpoint always works — the auto-remediation escape hatch.
