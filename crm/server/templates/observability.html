360 Correlation Guide

How to correlate signals across pillars in OCI Log Analytics

1

APM Trace Explorer

Find a slow or errored trace. Copy the trace_id.

2

Log Analytics

Search: 'trace_id' = '<paste-id>' — see all logs, security events, and sync audit entries for that request.

3

DB Management / OPSI

The trace_id is set as CLIENT_IDENTIFIER on Oracle sessions. In DB Management SQL Monitor, filter by client_identifier to see SQL performance for that request.

4

WAF Correlation

WAF headers (x-oci-waf-score, x-oci-waf-action) are captured as span attributes and logged as security events. Search Log Analytics for 'security.attack.type' = 'security_misconfig'.

5

Cross-Service

CRM propagates traceparent and X-Correlation-Id to Drone Shop. Both services' logs share the same trace_id for distributed request flows.
