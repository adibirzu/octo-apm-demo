# Milestone v1.1 — scaling-demo (Archived)

## Goal

Demonstrate OCI-native cluster elasticity end-to-end. Wire Horizontal Pod Autoscaler (HPA) + Cluster Autoscaler so the OKE deployment scales pods + nodes up under load and back down when traffic returns to normal. Ship an admin-only stress-test entry point that ramps load against shop + CRM, then surface the scaling story through OCI APM + OCI Monitoring + OCI Log Analytics in parallel.

## Key accomplishments

1. **End-to-end OCI scaling story shipped.** From traffic trigger (admin-only stress UI) through HPA (CPU/memory/RPS) to Cluster Autoscaler (worker node scale), with three observability surfaces (APM service map, OCI Monitoring custom metrics + alarms, Log Analytics saved searches + dashboard) each capturing a different facet of the scale narrative.
2. **Host-bound admin enforcement extracted to a single source.** `_admin_host.require_admin_host` lives once in `crm/server/modules/_admin_host.py`, imported by both `coordinator.py` (Phase 5) and the new `stress_test.py` — drift is structurally impossible. Phase 5 admin-host regression suite (`tests/test_admin_host_helper.py` + `crm/tests/test_admin_coordinator.py`) stayed green.
3. **k6 + FastAPI internal control plane for synthetic load.** Long-lived `octo-stress-runner` Deployment (`octo-stress` namespace) exposes 4-endpoint internal API gated by `X-Internal-Service-Key`, concurrency=1 enforced via asyncio.Lock + SIGTERM drain, 3 k6 scenarios (checkout journey / catalog browse / login burst) carrying `X-Octo-Stress-Target: oke` + `X-Run-Id` headers.
4. **Backward compatibility preserved by default.** All new HPA + stress assets are gated off by default (`autoscaling.rps.enabled=false`, `stressRunner.enabled=false`). `helm template` with defaults produces 0 External-metric blocks; only `--set` flips render the new surfaces. Phase 4 deployment-parity contract intact.
5. **D-21 OTel agent pins refreshed.** Java OTel agent 2.27.0 + LLMetry pin file shipped; existing OBS-01..05 field-shape contract verified via `shop/tests/test_llmetry.py` 2/2.
6. **Operator-deferred items documented, not gated.** 5 items needing live OCI Console + cluster credentials (CA add-on apply, LB header-routing rule, prometheus-adapter install, `apply.sh APPLY=true` runs, Lab 11 walkthrough) remain on the live-rollout punch list per STATE.md Blockers/Concerns.
