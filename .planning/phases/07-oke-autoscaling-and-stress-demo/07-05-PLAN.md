    The CRM Admin FastAPI gains four JSON endpoints under `/api/admin/stress/`
    plus one HTML page route `/admin/stress-test`. Every endpoint requires
    (a) admin role via `require_admin_user`, (b) admin host via
    `_require_admin_host`. Server-side hard caps reject out-of-range params
    with HTTP 422 before any side effect. Successful `/apply` generates a
    UUID `run_id`, opens an OTel span `admin.stress.apply` with attributes
    including `stress.run_id`, calls `push_log("INFO", "stress_test.apply",
    run_id=..., ...)` with the full D-15 MELTS field set, calls
    `oci_monitoring.increment_stress_run(run_id=..., status="started")`,
    then POSTs to the stress-runner pod's `/internal/run` endpoint with the
    `X-Internal-Service-Key` header sourced from cfg. On HTTP 503 from the
    runner, surface the runbook copy from UI-SPEC. On HTTP 409 from runner,
    re-raise as 409 with the active `run_id`. `/clear` mirrors `/apply` with
    `status="stopped"`. `/state` queries the runner's `/internal/state` and
    returns a sanitized projection (omit pod internals; surface only the
    fields the UI panel needs per UI-SPEC layout). The page route
    `/admin/stress-test` renders `stress_test_admin.html` (authored by
    Plan 07-06) with template context including `nav_key="stress"` so the
    sidebar entry highlights when the operator is on the page.
