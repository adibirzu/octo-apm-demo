# Phase 7 — UI Design Contract

> Visual and interaction contract for the admin-only `/admin/stress-test` surface
> on `admin.${DNS_DOMAIN}`. Extends the locked Phase 5 admin design language —
> does NOT redesign it. Generated by gsd-ui-researcher; verified by gsd-ui-checker.

## Existing System Reuse (Source of Truth)

Phase 7 is admin-only. The Phase 5 admin surface (CRM/Admin FastAPI under
`crm/server/templates/`) already locks every visual and interaction token this
phase needs. **D-11 makes this explicit:** `stress_test_admin.html` mirrors
`chaos_admin.html`. The contract below is a re-statement of what already ships
in `crm/server/static/css/style.css` and `crm/server/templates/base.html`,
applied to the new page.

| Surface | Reused From | Notes |
|---------|-------------|-------|
| Sidebar nav + header | `crm/server/templates/base.html` | Stress Test entry added to nav menu (admin-only nav block) |
| Page shell (header + body) | `base.html` block layout | `{% extends "base.html" %}` |
| Card / panel | `.card` in `style.css:233-298` | Glass surface, 26px radius, cyan stroke |
| Buttons | `.btn`, `.btn-primary`, `.btn-danger` in `style.css:569-617` | Pill (999px), 42px min-height, weight 600 |
| Form inputs | `.create-form label` pattern in `style.css:725-731` | Already used by `chaos_admin.html` |
| Code/JSON block | `pre`, `.code-output` in `style.css:658-669` | IBM Plex Mono, 18px radius |
| Status pill | `.badge`, `.badge-active`, `.badge-cancelled` in `style.css:619-656` | Reused for run status |
| Page header copy | `.page-header h1 + .page-subtitle p` (chaos_admin.html:5-12) | Same tone, same audit-banner pattern |
| Eyebrow + telemetry pill | `.eyebrow`, `.telemetry-pill` in `style.css:85-106` | Reused for "Admin only — audited" banner |

**No new design tokens, fonts, color rules, or component primitives are
introduced.** The stress-test page is a new instance of the existing
glass-dark admin language.

Declared values (extracted from existing `style.css`; already 4-point aligned):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline icon/text gaps (`gap: 4px` in nav-menu) |
| sm | 8px | Compact element gaps (`gap: 8px`, badge padding) |
| md | 16px | Default element gaps (form rows, `.page-container gap: 16px`) |
| lg | 24px | Card inner padding (`.card padding: 24px`) |
| xl | 32px | Page header horizontal padding (`.content header: 28-32px`) |
| 2xl | 48px | Page bottom padding (`.content padding: 28px 32px 48px`) |

the existing repo-wide convention from `style.css:579`. Do not change it.

is intentional and already shipped repo-wide.

## Typography

Sizes taken verbatim from `style.css`; no new sizes introduced. Weight is
either inherited (400 by default) or `600` for buttons / strong UI labels
(already declared in `.btn font-weight: 600`).

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 0.95rem (~15px) | 400 | 1.5 |
| Label / form caption | 0.85rem (~14px) | 400 | 1.4 |
| Heading (h2 — card section) | 1.15rem (~18px) | 700 | 1.3 |
| Display (h1 — page title) | 2.35rem (~38px) | 700 | 1.15 |
| Code / audit JSON | 0.82rem (~13px) | 400 | 1.5 |
| Eyebrow / pill | 0.72rem (~12px) | 400 | 1.2 (uppercase, letter-spacing 0.16em) |

Total **5 sizes including code/eyebrow**; **2 weights** (400 + 700, with 600
reserved for the existing `.btn` rule). This exceeds the 3–4 sizes / 2 weights
heuristic, but the additional roles (code monospace + eyebrow) are
repo-locked from Phase 5 and not re-asked.

## Color

60 / 30 / 10 split derived from the existing palette declared in
`style.css:1-18`. No new color is introduced.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--bg-base #07111c` + radial cyan glow gradient | Page background, body fill |
| Secondary (30%) | `--bg-panel rgba(10, 28, 43, 0.88)` | Cards, panels, sidebar surface |
| Accent (10%) | `--accent #75d9e3` / `--accent-strong #8ceaf0` | Eyebrow, primary CTA gradient, telemetry-on pill, active nav, link underlines, focus rings |
| Destructive | `--danger #ff7474` | Stop run button, error pill, audit `status: error` badge |
| Success | `--success #65d4a8` | `status: started/running` badge |
| Warning | `--warning #f3c76f` | `status: expired/stopped` badge (informational, not red) |

**Accent reserved for:**

1. Primary CTA "Apply stress run" button background gradient.
2. Active nav-item highlight in the sidebar.
3. `.eyebrow` text and audit-banner accent stroke at the top of the page.
4. "RUN ID" copyable code chip border.
5. Outbound drilldown link borders (APM, Logan, Monitoring, external
   observability hosts — D-20).

Accent is **not** used for body text, every secondary button, or every
form input — those use the neutral `--text-secondary` / `--bg-panel` tokens.

**Destructive** (`--danger`) is reserved for:

1. The "Stop active run" button (`.btn-danger`).
2. Run-failed badge.
3. Server-side error result block (`#apply-result` text when status code != 200).

## Copywriting Contract

Phase 5 already locks tone guardrails: avoid backend / internal wording, avoid
live tenancy labels, all admin actions must surface an audit banner. The
stress-test page extends that contract.

### Page chrome

| Element | Copy |
|---------|------|
| Sidebar nav label | `Stress Test` |
| Page eyebrow | `OCI Observability demo project` (inherited from `base.html`) |
| Page H1 (`title`) | `OKE stress test` |
| Page subtitle | `Restricted to the admin role (optional stress-operator scope). Triggers a bounded k6 load run against the demo storefront. Every run is audited, capped, and stoppable.` |

### Form fields and primary CTA

| Element | Copy |
|---------|------|
| Section heading: "Currently active" | `Active stress run` |
| Section heading: "Apply a preset" | `Trigger a new run` |
| Scenario selector label | `Scenario` |
| Target service label | `Target service` (single option `shop` per D-13, but field stays visible for clarity) |
| RPS field label | `Requests per second (1–200)` |
| Duration field label | `Duration (seconds, 10–600)` |
| Run note field label | `Run note (who / why)` |
| Primary CTA button | `Apply stress run` |
| Active-run stop button | `Stop active run` |
| Empty active-run heading | `No stress run active` |
| Empty active-run body | `Trigger a run below. The k6 wrapper enforces concurrency=1 and a server-side hard timeout of duration + 30s.` |

### State copy

| State | Copy |
|-------|------|
| Run starting (HTTP 202) | `Run {run_id} started — {rps} RPS for {duration}s against {target_host}` |
| Run already in progress (HTTP 409) | `A stress run is already active: {run_id} (started {timestamp_start}). Stop it before starting another.` |
| Validation error (HTTP 422) | `Request rejected: {field} must be between {min} and {max}. Hard caps are enforced server-side.` |
| Auth / host error (HTTP 401/403) | `Stress test access is restricted to admin role on admin.{DNS_DOMAIN}.` |
| Wrapper unreachable (HTTP 503) | `Stress runner unavailable. Confirm octo-stress-runner Deployment is healthy in the OKE cluster.` |
| Run stopped by operator | `Run {run_id} stopped at {timestamp_end}.` |
| Run auto-expired | `Run {run_id} completed at {timestamp_end} (duration reached).` |

### Audit banner (top-of-page, always visible)

Copy: `Admin only — every action is audited. Audit fields include trace_id,
span_id, run_id, admin_user, admin_role, rps_requested, duration_requested,
scenario, target_service, status, and reason.`

### Destructive confirmation

| Action | Confirmation copy |
|--------|--------------------|
| Stop active run | `Stop run {run_id}? k6 receives SIGTERM and drains in-flight requests. The audit event will record status=stopped with the current operator.` |

Confirmation pattern: a `confirm()` dialog before the DELETE call (matches
the existing `chaos_admin` `Clear scenario` pattern, which currently fires
immediately without a confirm; Phase 7 ADDS a confirm step for the stop
action because a stress run is more disruptive than a chaos preset).

### Forbidden copy

Per Phase 5 lock:

- Do NOT name backend services using internal wording (e.g. avoid
  "Workflow Gateway", "Coordinator", "k6 wrapper container" in operator-facing
  text — keep them in the audit JSON and runbooks).
- Do NOT print the live OCI tenancy OCID, compartment OCID, or worker pool
  OCID in the UI. The UI only displays the public `target_host` (e.g.
  `shop.octodemo.cloud`) and the `run_id`.
- Do NOT include real customer email addresses or live operator names in
  copy examples; placeholders only.

## Interaction Contract

### Layout

```
┌────────────────────────────────────────────────┐
│  [eyebrow]                       [header_actions: link to APM, Logan]
│  OKE stress test                                │
├────────────────────────────────────────────────┤
│  Audit banner (cyan stroke, eyebrow tone)       │
│                                                 │
│  ┌─ Active stress run ─────────────────────┐   │
│  │  status: [badge]   run_id: [chip]        │   │
│  │  scenario · rps · duration · started_at  │   │
│  │  <pre>full audit JSON</pre>              │   │
│  │  [ Stop active run ] (danger, confirm)   │   │
│  └──────────────────────────────────────────┘   │
│                                                 │
│  ┌─ Trigger a new run ──────────────────────┐   │
│  │  Scenario [select]                        │   │
│  │  Target   [select disabled]               │   │
│  │  RPS      [number 1..200]                 │   │
│  │  Duration [number 10..600]                │   │
│  │  Note     [text 0..512]                   │   │
│  │  [ Apply stress run ] (primary)           │   │
│  │  <pre id="apply-result"></pre>            │   │
│  └──────────────────────────────────────────┘   │
│                                                 │
│  ┌─ Observability drilldowns ───────────────┐   │
│  │  APM saved queries (4) · Monitoring     │   │
│  │  alarms (2) · Logan dashboard           │   │
│  │  · lm/phoenix/openlit/grafana .octodemo  │   │
│  └──────────────────────────────────────────┘   │
└────────────────────────────────────────────────┘
```

### Polling

- `Active stress run` panel polls `GET /api/admin/stress/state` every **2s**
  while a run is active, every **10s** while idle. Mirrors the
  `chaos_admin.html` refresh cadence but adds short polling during an
  active run so the operator sees `status: running → expired/stopped`
  transitions in real time.
- During an active run, the panel also surfaces the **current shop pod
  count** retrieved from `/api/admin/stress/state.pod_count` (server fetches
  from `octo_apm_demo_shop_pod_count` published metric per D-17). This is the
  on-page signal that HPA is firing; the deep observability drilldown lives in
  APM/Logan (D-16 / D-19).

### Hover / focus / active states (reused from existing CSS)

- Buttons: `:hover` translates `-1px` (already in `.btn:hover` rule).
- Nav items: `:hover/.active` adds cyan stroke + translates `+2px`.
- Form inputs inherit `.create-form label` / global input styling from
  `style.css` (no new focus ring introduced).
- Active run badge animates a soft cyan pulse (CSS-only, compositor-friendly:
  `opacity` + `transform: scale()` keyframes, respects
  `prefers-reduced-motion`).

### Reduced motion

- `prefers-reduced-motion: reduce` MUST disable the pulse animation on the
  active-run badge and the button `translateY` hover. This matches the
  global web/performance rule already enforced repo-wide.

### Keyboard / a11y

- Tab order: scenario → target → rps → duration → note → Apply.
- Stop button reachable independently in the active-run section.
- All form labels are `<label>` elements wrapping their input (already the
  pattern in `chaos_admin.html`).
- Audit `<pre>` is `role="status" aria-live="polite"` so screen readers
  hear run state changes.
- No focus traps; no modal dialogs (stop confirm uses native `confirm()`).
