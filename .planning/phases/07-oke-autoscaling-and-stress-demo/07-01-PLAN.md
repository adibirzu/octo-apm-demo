<objective>
Expand HPA scope to cover `octo-drone-shop` (maxReplicas 4→10) and add a new
HPA for `octo-apm-java-demo` (maxReplicas 6). Both HPAs combine CPU + memory
+ an opt-in External RPS metric (D-02, D-03). All scaling changes are
additive and backward-compatible: the RPS metric is gated behind a Helm
value `autoscaling.rps.enabled` that defaults to `false` (D-05) so existing
`helm template` output is bit-identical until an operator flips the flag.
CRM HPA is intentionally untouched (D-01).

This plan is the SOLE owner of `deploy/helm/octo-apm-demo/values.yaml` in
Wave 1. It also seeds the top-level `stressRunner:` values block (default
`enabled: false`) that Plan 07-03 templates reference — so Plan 07-03 can
ship templates without re-touching values.yaml.

This plan also lands the D-21 dependency bump: Python OTel SDK +
instrumentation pins, OTel Java agent, and LLMetry pins are advanced to
current stable; existing OBS-01..05 field-shape regression tests still
pass.

Purpose: Provide the pod-level elasticity that drives the Phase 7 demo
narrative — when stress traffic ramps, HPA observes RPS and adds shop pods
up to the new max of 10, then Cluster Autoscaler (plan 02) reacts to add
nodes when scheduling pressure hits. Keep the dependency surface current so
later phases inherit fixed CVEs and instrumentation improvements without
breaking the field-shape contract.

Output: Raw OKE HPA manifest edits for shop + java; Helm template edits +
new java-gateway-hpa.yaml; values.yaml additions including stressRunner
block; OTel/LLMetry pin bumps + a regression test. No live apply.
</objective>
