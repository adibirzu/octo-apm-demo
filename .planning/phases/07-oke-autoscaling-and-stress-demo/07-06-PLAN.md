<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → admin template render | Server-rendered Jinja2 — no client-controlled context |
| Browser → inline JS | CSP-nonced; nonce is per-request from `request.state.csp_nonce` |
| Inline JS → /api/admin/stress/* | Same-origin fetch; admin host enforcement in API layer |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-31 | Tampering | XSS via run_id or note rendered in `<pre>` | mitigate | `<pre>` uses `textContent` not `innerHTML` in inline JS; Jinja2 default auto-escapes server-rendered template variables |
| T-07-32 | Tampering | External JS injection bypassing CSP | mitigate | Template asserts no `<script src=`; CSP nonce mechanism enforces; test `test_stress_template_no_external_script_tags` regression |
| T-07-33 | Information Disclosure | Forbidden copy leaks internals (Workflow Gateway, Coordinator) | mitigate | UI-SPEC forbidden-copy list enforced via `test_stress_template_no_internal_wording`; only audit `<pre>` and `<script>` may reference internal names |
| T-07-34 | Information Disclosure | Live OCID/IP leaks in template | mitigate | `test_stress_template_no_live_ocids` grep gate; all environment-derived values (`dns_domain`, `drilldown_links`) come from cfg, never hardcoded |
| T-07-35 | Repudiation | Operator stops a run without confirmation | mitigate | `confirm()` step with run_id substitution; copy explicitly says audit will record status=stopped |
</threat_model>
