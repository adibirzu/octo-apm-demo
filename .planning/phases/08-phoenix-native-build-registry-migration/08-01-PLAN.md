<objective>

Establish reachability to the Phoenix build host `octo-emdemo-jumphost-v5` and create the inventory-capture skeleton that Plans 08-02 and 08-03 will extend. This plan must NOT install tools or mutate the NSG — it only verifies SSH works and reports the gap so the operator can decide whether to add a `/32` ingress rule manually.

This plan addresses **BUILD-02** (reproducible build pipeline on Phoenix jumphost) by getting the operator into a known reachable state and capturing the jumphost baseline (arch, disk, existing tool inventory) before any later plan installs software. All NSG mutation is operator-gated (the script prints commands but never executes `oci ... add` itself), per `.planning/SECURITY.md` and the `<security_rules>` for this phase.
