  For each canary step:
  ```bash
  cutover_one() {
      local svc="$1" deploy="$2" ns="$3" container="$4" repo="$5"
      local image="${OCIR_REGION}.ocir.io/${OCIR_TENANCY}/${repo}:${IMAGE_TAG}"
      echo "── Canary: ${svc} ─────────────────────────────────────"
      if [[ "${APPLY}" != "true" ]]; then
          echo "DRY-RUN: would run: kubectl -n ${ns} set image deployment/${deploy} ${container}=${image}"
          return
      fi
      kubectl -n "${ns}" set image "deployment/${deploy}" "${container}=${image}"
      kubectl -n "${ns}" rollout status "deployment/${deploy}" --timeout=10m
      probe_apm_continuity "${svc}" 300 30  # window_seconds=300, interval_seconds=30
  }
  ```

  APM continuity probe (`probe_apm_continuity`) — heart of BUILD-04:
  - Inputs: just-cut-over service name, window seconds (default 300 = 5 min), poll interval
    (default 30s).
  - Expected service set (per 08-CONTEXT.md success criteria #4, "all 8 services healthy"):
    `octo-drone-shop`, `octo-apm-java-demo`, `octo-workflow-gateway`, `enterprise-crm-portal`,
    `octo-traffic-generator`, `octo-java-app-server-oke`, `octo-frontend-ingress`,
    `octo-stress-runner`. (Executor verifies the exact 8 names against the live APM dashboard
    before hardcoding — adjust if APM uses different labels in this tenancy.)
  - Implementation: use `oci apm-traces query` or `oci monitoring metric-data summarize-metrics-data`
    (the project already uses oci-cli for APM in `tools/apm-saved-queries/apply.sh`). For each
    poll, request the last 60 seconds of "service.health" or equivalent metric for the 8 services.
    Track the most recent "seen" timestamp per service. If any service's last-seen lag exceeds
    60 seconds at any point in the window, FAIL the probe (exit non-zero — cutover aborts).
  - The exact OCI CLI invocation depends on the APM SDK shape; the executor should:
    a) First check if there is an existing helper in `tools/apm-saved-queries/` or
       `tools/monitoring-alarms/` that already polls APM — reuse it.
    b) If not, write a minimal `query_apm_service_seen` helper inside this same script using
       `oci monitoring metric-data summarize-metrics-data --compartment-id "${COMPARTMENT_ID}"
       --namespace oci_apm --query-text 'SystemMetrics[1m]{ServiceName="<svc>"}.mean()' --start-time
       "$(date -u -d '60 seconds ago' +%FT%TZ)" --end-time "$(date -u +%FT%TZ)" --profile "${OCI_PROFILE}"`.
       Empty result for a service → that service is silent → count toward the 60s gap budget.
    c) Print a one-line status per poll: `[+60s] traffic-gen ✓ java-demo ✓ workflow ✓ shop ✓ crm ✓
       (5/8 reporting)` so the operator sees the probe live.
  - On FAIL, the probe MUST `set -e` propagate so the outer script aborts WITHOUT running the next
    canary step. Print a recovery hint: "APM continuity broken on <svc>. Run: kubectl -n <ns>
    rollout undo deployment/<deploy>".

  Idempotency:
  - Before each `kubectl set image`, read the current image with
    `kubectl -n "${ns}" get deploy "${deploy}" -o jsonpath='{.spec.template.spec.containers[?(@.name=="'"${container}"'")].image}'`.
    If it already equals the target image, skip with `echo "  ${svc}: already at target image, skipping."`.

  Helm rollback escape hatch (BUILD-06 hook — fully exercised in 08-08):

|----------|-------------|
| operator → OKE control plane | kubectl set image mutates live workloads — credential exposure risk if KUBECONFIG leaks |
| operator → OCI APM API | APM query needs read access to the LogAnalytics compartment metrics |
| OKE pod → phx OCIR | image pull crosses cluster boundary; pull secret must be present in target namespace (already seeded by Phase 6) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-07-01 | Tampering | rolling-cutover.sh | mitigate | APPLY=false default + interactive confirm phrase prevents accidental cutover. Script refuses if `OCIR_REGION` != `us-phoenix-1` without explicit override. |
| T-08-07-02 | Denial of service | live cluster | mitigate | maxUnavailable=0 in all 5 deployments (already in manifests), `kubectl rollout status --timeout=10m` gates each step, APM continuity probe aborts on >60s gap. |
| T-08-07-03 | Information disclosure | script logs | mitigate | Script reads `APM_DOMAIN_ID`, `COMPARTMENT_ID`, `OCIR_TENANCY` from env / outputs.json — never inlines. Logs print `${OCIR_TENANCY}` as a literal env-var name, not its resolved value. |
| T-08-07-04 | Repudiation | post-cutover audit | mitigate | Script prints `helm history octo-apm-demo` revision number at the end, operator copies it into the SUMMARY — establishes the rollback target. |
