<objective>

Create the operator-gated docker login wrapper `deploy/oke/ocir-login.sh` that authenticates the Phoenix jumphost to `${OCIR_REGION}.ocir.io` using an Auth Token read from `~/.ocir-token` (mode 0600), piped to docker via `--password-stdin`. Pair it with a runbook documenting the Auth Token generation procedure and bash-history hygiene.

This plan addresses **BUILD-02** (auth via Auth Token, never API key; token piped via stdin to keep it out of shell history) and is the last prerequisite before the build pipeline pilot in Plan 08-04. Per `<security_rules>` and PATTERNS shared pattern D, the token MUST NEVER appear as a `--password <val>` argument or be assigned to a shell variable that could be echoed. The runbook MUST use placeholder tokens only (`<OCIR_USERNAME>`, `${OCIR_TENANCY}`) per `.planning/SECURITY.md`.

</objective>
